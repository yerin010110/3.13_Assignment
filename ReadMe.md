# 1. 강화학습 개요

강화학습(Reinforcement Learning)은 에이전트(agent)가 환경(environment)과 상호작용하면서 보상을 최대화하는 행동 전략을 학습하는 방법입니다. 일반적인 지도학습(supervised learning)은 입력 데이터와 정답 데이터가 함께 주어지지만 강화학습에서는 정답이 존재하지 않습니다. 대신 에이전트는 환경과 반복적으로 상호작용하면서 보상을 관찰하고 그 보상을 최대화하는 방향으로 정책을 개선합니다.

강화학습은 일반적으로 **Markov Decision Process(MDP)** 로 모델링됩니다. MDP는 의사결정을 포함하는 확률적 모델이며 다음과 같은 구성 요소를 포함합니다.

State(상태)는 환경의 현재 상황을 나타냅니다. 예를 들어 CartPole 환경에서는 카트의 위치, 속도, 막대의 각도, 막대의 각속도 등이 상태를 구성합니다.  

Action(행동)은 에이전트가 선택할 수 있는 행동을 의미합니다. CartPole 환경에서는 카트를 왼쪽으로 이동하거나 오른쪽으로 이동하는 두 가지 행동이 존재합니다.  

Reward(보상)는 에이전트가 특정 행동을 수행한 결과로 환경으로부터 받는 신호입니다. 강화학습에서는 이 보상을 최대화하는 방향으로 학습이 진행됩니다.  

Transition(상태 전이)은 특정 상태에서 특정 행동을 했을 때 다음 상태로 이동하는 과정을 의미합니다. 이 과정은 확률적으로 결정될 수 있습니다.  

Discount Factor(할인율 γ)는 미래 보상을 얼마나 중요하게 고려할지를 결정하는 파라미터입니다.

강화학습에서 중요한 개념 중 하나는 **Return** 입니다. Return은 특정 시점 이후에 얻을 수 있는 보상의 총합을 의미합니다. Return은 다음과 같은 수식으로 표현됩니다.

Gₜ = Rₜ + γRₜ₊₁ + γ²Rₜ₊₂ + ...

이 수식은 현재 시점 이후에 얻을 수 있는 보상들을 모두 합산한 값이며 미래 보상은 할인율 γ에 의해 점점 작아집니다. 할인율이 1에 가까울수록 미래 보상을 중요하게 고려하며 0에 가까울수록 현재 보상을 더 중요하게 고려합니다.

강화학습 알고리즘의 목표는 이러한 Return을 최대화하는 정책을 학습하는 것입니다. 즉 에이전트는 현재 상태에서 어떤 행동을 선택해야 장기적으로 가장 높은 보상을 얻을 수 있는지를 학습하게 됩니다.

# 2. Policy Gradient 방법

강화학습 알고리즘은 크게 **Value 기반 방법**과 **Policy 기반 방법**으로 나눌 수 있습니다. Value 기반 방법은 상태 또는 행동의 가치를 계산하여 가장 높은 가치를 가지는 행동을 선택하는 방식입니다. 대표적인 알고리즘으로는 Q-Learning과 Deep Q-Network(DQN)이 있습니다.

반면 Policy 기반 방법은 정책(policy)을 직접 학습합니다. 정책은 특정 상태에서 어떤 행동을 선택할 확률을 나타내는 함수입니다. 즉 Value 기반 방법이 행동의 가치를 계산한 후 행동을 선택하는 방식이라면 Policy 기반 방법은 행동 확률 자체를 학습합니다.

대표적인 Policy Gradient 알고리즘으로는 **REINFORCE**가 있습니다. REINFORCE는 정책의 파라미터를 직접 업데이트하여 기대 보상을 최대화하도록 학습합니다. Policy Gradient의 기본 아이디어는 다음과 같습니다.

정책 π(a|s)가 있을 때 특정 상태에서 특정 행동을 선택할 확률을 나타냅니다. Policy Gradient 방법은 정책의 파라미터 θ를 조정하여 기대 보상을 증가시키는 방향으로 학습합니다.

하지만 REINFORCE와 같은 기본적인 Policy Gradient 알고리즘은 **분산(variance)이 매우 크다는 문제**가 있습니다. 즉 학습 과정에서 결과가 매우 불안정하며 수렴 속도가 느립니다. 이러한 문제는 강화학습에서 매우 중요한 문제로 알려져 있습니다.

이러한 문제를 해결하기 위해 등장한 방법이 **Actor-Critic 구조**입니다. Actor-Critic 방법은 정책을 학습하는 Actor와 상태 가치를 평가하는 Critic을 동시에 사용하여 Policy Gradient의 분산 문제를 줄이고 보다 안정적인 학습을 가능하게 합니다.

# 3. Actor-Critic 구조

Actor-Critic 알고리즘은 강화학습에서 매우 중요한 구조 중 하나입니다. 이 방법은 두 개의 신경망을 사용합니다. 하나는 **Actor**, 다른 하나는 **Critic** 입니다.

Actor는 정책(policy)을 학습하는 역할을 합니다. 즉 현재 상태가 주어졌을 때 어떤 행동을 선택할지를 확률적으로 결정합니다. Actor는 일반적으로 정책 네트워크(policy network)라고도 불립니다.

Critic은 상태의 가치를 평가하는 역할을 합니다. 즉 특정 상태가 얼마나 좋은 상태인지 Value Function을 통해 추정합니다. Critic은 Value Network라고도 불립니다.

Actor와 Critic은 서로 다른 역할을 수행하지만 동시에 학습됩니다. Actor가 행동을 선택하면 Critic은 그 행동이 얼마나 좋은 행동인지 평가합니다. Critic이 계산한 가치 정보는 Actor의 정책 업데이트에 사용됩니다.

Actor-Critic 구조의 가장 큰 장점은 Policy Gradient 방법의 분산 문제를 줄일 수 있다는 점입니다. Critic이 상태 가치 정보를 제공함으로써 정책 업데이트 과정이 보다 안정적으로 이루어집니다.

또한 Actor-Critic 방법은 Value 기반 방법과 Policy 기반 방법의 장점을 동시에 활용할 수 있습니다. Value 기반 방법의 안정성과 Policy 기반 방법의 확률적 정책 학습 능력을 결합한 형태라고 볼 수 있습니다.

# 4. Advantage 개념

Actor-Critic 알고리즘에서 중요한 개념 중 하나는 **Advantage** 입니다. Advantage는 특정 행동이 평균적인 행동보다 얼마나 더 좋은지를 나타내는 값입니다.

Advantage는 다음과 같은 수식으로 정의됩니다.

A(s,a) = Q(s,a) − V(s)

여기서 Q(s,a)는 상태 s에서 행동 a를 수행했을 때 얻을 수 있는 기대 보상이며 V(s)는 상태 s의 평균적인 가치입니다.

Advantage 값이 양수이면 해당 행동이 평균보다 좋은 행동이라는 의미이며 정책 업데이트 과정에서 그 행동의 확률이 증가합니다. 반대로 Advantage 값이 음수이면 해당 행동은 평균보다 나쁜 행동이므로 정책 업데이트 과정에서 그 행동의 확률이 감소합니다.

Advantage 개념을 사용하면 정책 업데이트 과정에서 불필요한 변동을 줄이고 실제로 좋은 행동만 강화하는 효과를 얻을 수 있습니다. 따라서 Advantage는 Actor-Critic 기반 강화학습 알고리즘에서 매우 중요한 역할을 합니다.

# 5. A2C 알고리즘

A2C(Advantage Actor-Critic)는 Actor-Critic 구조에 Advantage 개념을 적용한 강화학습 알고리즘입니다. A2C는 A3C(Asynchronous Advantage Actor-Critic) 알고리즘의 동기식 버전입니다.

A3C 알고리즘은 여러 환경을 동시에 실행하여 학습 속도를 높이는 방법입니다. 하지만 구현이 복잡하며 시스템 자원을 많이 사용합니다. 이에 비해 A2C는 하나의 환경에서 동기적으로 학습을 진행하기 때문에 구현이 비교적 단순하며 안정적인 학습이 가능합니다.

A2C의 학습 과정은 다음과 같은 단계로 이루어집니다.

1. 현재 상태를 Actor 네트워크에 입력하여 행동 확률을 계산합니다.  
2. 확률 분포에서 행동을 샘플링합니다.  
3. 선택된 행동을 환경에 적용합니다.  
4. 환경으로부터 다음 상태와 보상을 받습니다.  
5. Critic 네트워크를 이용하여 상태 가치 V(s)를 계산합니다.  
6. TD Target을 계산합니다.  
7. Advantage를 계산합니다.  
8. Actor와 Critic 네트워크를 업데이트합니다.

TD Target은 다음과 같이 계산됩니다.

TD Target = r + γV(s')

Advantage는 다음과 같이 계산됩니다.

Advantage = TD Target − V(s)

Actor는 Advantage를 이용하여 정책을 업데이트하며 Critic은 Value Function의 오차를 최소화하도록 학습합니다.

# 6. 실험 환경 (CartPole)

본 프로젝트에서는 Gymnasium의 **CartPole-v1** 환경을 사용합니다. CartPole 문제는 강화학습 알고리즘을 테스트하기 위해 널리 사용되는 대표적인 환경입니다.

이 환경에서는 카트 위에 세워진 막대가 넘어지지 않도록 카트를 좌우로 움직여 균형을 유지해야 합니다. 에이전트는 막대가 최대한 오래 넘어지지 않도록 행동을 선택해야 합니다.

CartPole 환경의 상태는 다음 네 가지 변수로 구성됩니다.

- Cart Position (카트 위치)  
- Cart Velocity (카트 속도)  
- Pole Angle (막대 각도)  
- Pole Angular Velocity (막대 각속도)

행동은 다음 두 가지입니다.

- 0 : 카트를 왼쪽으로 이동  
- 1 : 카트를 오른쪽으로 이동  

에이전트가 막대를 넘어뜨리지 않고 균형을 유지할수록 더 높은 보상을 받게 됩니다. 에피소드가 진행될수록 에이전트는 막대를 더 오래 유지하는 방향으로 정책을 학습하게 됩니다.

CartPole 환경은 상태 공간이 비교적 단순하지만 강화학습 알고리즘의 핵심 동작을 이해하기에 매우 적합한 환경입니다.
